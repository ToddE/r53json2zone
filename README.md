# r53json2zone

A bash script that converts AWS Route 53 JSON exports to standard BIND zone files, making it easy to migrate DNS records to other providers (e.g. Cloudflare).


## What it does

- Converts Route 53 `ResourceRecordSets` JSON to BIND-format zone records
- Collapses AWS Alias records into standard `CNAME` entries for provider compatibility
- Detects advanced AWS routing features (Weighted, Geo, Failover, Health Checks) and warns you about what won't transfer automatically
- Optionally saves a CloudFlare-specific migration guide report alongside the zone file

## Requirements

- `bash`
- [`jq`](https://jqlang.github.io/jq/) (offered for auto-install if missing)

## Installation

### One-liner

```bash
curl -sSL https://raw.githubusercontent.com/ToddE/r53json2zone/main/install.sh | bash
```

This installs the script to `~/.local/bin/r53json2zone`. If that directory isn't in your `PATH`, add this to your `.bashrc` or `.zshrc`:

```bash
export PATH="$HOME/.local/bin:$PATH"
```

### Manual

Download [r53json2zone](r53json2zone), make it executable, and place it somewhere on your `PATH`:

```bash
chmod +x r53json2zone
mv r53json2zone ~/.local/bin/
```

## Usage

First, export your Route 53 hosted zone records to JSON using the AWS CLI:

```bash
aws route53 list-resource-record-sets \
  --hosted-zone-id <YOUR_ZONE_ID> \
  > records.json
```

Then convert:

```bash
r53json2zone <route53_json_file> <output_zone_file>
```

**Example:**

```bash
r53json2zone records.json example.com.zone
```

## Output

- `<output_zone_file>` — BIND-format zone file ready for import
- `info-<output_zone_file>` — (optional) Migration guide listing Alias-to-CNAME conversions and any advanced routing that needs manual attention

## Cloudflare notes

The migration report generated by this script includes Cloudflare-specific guidance. Key points:

- **CNAME at zone apex** — Cloudflare supports CNAMEs at the zone apex via CNAME Flattening, so Alias-to-CNAME conversion works without issue.
- **Advanced routing** — Weighted, latency, failover, and geo routing are AWS-specific features. Recreate them using Cloudflare Load Balancing or Traffic Steering.
- **Dual-stack** — Cloudflare's proxy serves both IPv4 and IPv6 from a single proxied CNAME. Separate `A` and `AAAA` records are not required once proxied.

## AWS CloudFront: Alternate Domain Names

If any records point to a CloudFront distribution, verify that the hostname is listed under the distribution's **Alternate Domain Names**. Requests for hostnames not listed there will return HTTP 403. This applies regardless of which DNS provider you migrate to.
